# Algorithm v0.7.0

**Commit:** `036eb53` | **Date:** 2026-04-20

## Summary

Replaced single 0-100 weighted score with a dual-score phase-based model. Burns now have **Potential** (site quality, stable) and **Readiness** (weather conditions, changes daily). Readiness is computed from a 44-day rolling timeline of START/GROW/BAD classifications, with coefficients learned via logistic regression on 70 labeled scenarios.

## Key characteristics

- **Dual scores:** Potential (0-100) + Readiness (0-100) replace single total
- **Phase labels:** EMERGING / GROWING / WAITING / TOO_EARLY
- **Timeline classification:** each of 44 days (30 hist + 14 forecast) tagged as START, START_GROW, GROW, or BAD
- **Readiness via regression:** 9 features, coefficients from 70 scenarios (21 synthetic + 49 real-world), 89% accuracy
- **Potential weighted 50% on burn quality** — burn age and type dominate site scoring
- **Anti-oscillation:** recovering from BAD can't immediately START
- **Rain triggers:** significant rain while in grow range counts as START_GROW
- Map: diamond shape = potential, size = 3 tiers, color = phase, number = readiness

## What changed from v0.6.1

- Replaced 6-factor weighted score (soil threshold + GDD + moisture + burn quality + sun/aspect + air temp) with Potential + Readiness
- Phase model classifies days as START/GROW/BAD based on daily conditions
- Readiness computed from rolling window: enough start days + enough grow days + no prolonged bad streaks
- Logistic regression on 70 labeled scenarios replaces hand-tuned weights
- Map markers: shape=potential, color=phase, number=readiness (was: color=score, number=score)
- Filters: potential + readiness sliders + raw condition sliders

## Known limitations

- Regression coefficients from 70 scenarios — more data (especially field reports) would improve accuracy
- `growth_was_reset` coefficient is positive (+0.85) — suspicious, likely a quirk of small sample
- Readiness saturates at 90%+ for many sites — logistic regression designed for classification not ranking
- `classify_day` start range (43-50F) may be too narrow — some sites never get classified as START
- No driving time estimation — straight-line distance misleads in mountains
- Old scoring code (`scoring.py`) still present alongside `phase_scoring.py` — needs cleanup
