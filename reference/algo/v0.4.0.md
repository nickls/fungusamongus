# Algorithm v0.4.0

**Commit:** `2ccad82` | **Date:** 2026-04-19

## Summary

First version with the soil temperature hard gate. Introduced the concept that cold soil should crush the entire score, not just lose points in one factor.

## Key characteristics

- **4-factor model:** temperature (25), moisture (25), burn quality (35), elevation (15) + terrain bonus (5)
- **Soil temp gate:** below 40F, all scores multiplied by 0.1
- **Warming trend:** first-half vs second-half mean comparison (later found to amplify noise from single-day spikes)
- **Temperature was a single combined factor** — soil threshold, air temps, and trend all lumped into one 25pt bucket
- **Burn quality dominated at 35pts** — later research showed temperature system should dominate timing

## What changed from v0.3.0

- Soil temp became a hard gate instead of a weighted factor
- Warming trend weight increased to 35% of temperature
- Recency curve refined: 0-2 month penalty for too-fresh burns
- Rating thresholds tightened: 80+ = excellent, <50 = hidden from map

## Known issues (fixed in later versions)

- Warming trend used half-split comparison, not linear regression — single-day spikes inflated scores
- Soil gate only applied to the warming_trend factor, not all factors
- 36-month-old burns still scored 7/35 because type+size bonus was added even when recency = 0
- Scored arbitrary named zones, not burn locations directly
