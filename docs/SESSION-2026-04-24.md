# Session Notes — April 24 – May 1, 2026

## Bugs Fixed

### 1. TOO_EARLY with readiness 90+ (the original bug)
`score_readiness()` (logistic regression) and `classify_phase()` (rule-based) were independent. The regression had a tiny coefficient (0.09) on `start_days`, so warm soil produced high readiness even with zero START events. 

**Fix:** `score_readiness()` now calls `classify_phase()` internally and caps output:
- TOO_EARLY → max 25
- WAITING → max 50
- GROWING/EMERGING → uncapped

*File: `phase_scoring.py:289-319`*

### 2. 69F soil classified as BAD ("past prime")
`grow_max` was hardcoded at 58F. Anything above was `BAD`. Field report FR-002 found 4-5lbs of morels at 69F soil.

**Fix:** Added `past_prime_max = 75F`. Soil 58-75F is `GROW` (past prime but harvestable). Only above 75F is `BAD`.

*File: `phase_scoring.py:40-41, 60-62, 88-91`*

### 3. Readiness crashes overnight (93 → 5)
Two causes:
- `is_currently_good` was binary (1 if today is GROW, 0 otherwise). Coefficient 1.04 = biggest feature. One BAD day → 30-point swing.
- `grow_days` reset to 0 on a 3-day bad streak, zeroing out biological progress.

**Fix:**
- `is_currently_good` is now a 5-day rolling ratio (e.g. 4/5 = 0.8 if 4 of last 5 days are good)
- Added `grow_days_total` (never resets) for readiness. `grow_days_since_reset` used only for phase classification.

*File: `phase_scoring.py:173-248`*

### 4. Detail page URLs break on every data refresh
`detail.html?i=25` used array index. Re-sorting or adding sites changed what index 25 points to.

**Fix:** Added URL slugs (e.g. `?site=waddle-ranch-rx-2024`). Generated in `build_sites.py`, exported through `morel_finder.py`, used in `app.js` and `detail.html`. Old `?i=` links still work as fallback.

*Files: `build_sites.py`, `morel_finder.py`, `docs/app.js:363`, `docs/detail.html:66-88`*

### 5. Sites hidden from map even with sliders at zero
Filter defaults were `potential: 50, readiness: 25`. These applied on page load regardless of slider position.

**Fix:** Set both defaults to 0.

*File: `docs/app.js:9-10`*

### 6. TOO_EARLY markers colored orange (same as WAITING)
Map marker color fell through to default `"orange"` for TOO_EARLY.

**Fix:** Use `phaseColorMap[phase]` which maps TOO_EARLY to gray.

*File: `docs/app.js:287`*

### 7. Timeline tooltips broken by HTML entities
Reason strings like `"too cold (39F < 43F)"` had `<` which broke the `title` attribute.

**Fix:** Changed reason format to `"too cold (39F, need 43F+)"` and added HTML escaping in the JS.

*Files: `phase_scoring.py:50-54`, `docs/detail.html:275`*

### 8. Python 3.8 compat (`dict | None` syntax)
Multiple utils files used `dict | None` union syntax (requires 3.10+). The venv is 3.14 but shell alias pointed to 3.8.

**Fix:** Added `from __future__ import annotations` to `utils/cache.py`, `http.py`, `elevation.py`, `pfirs.py`, `scoring.py`.

## Architecture Changes

### Static site catalog (`data/sites.json`)
Previously every run fetched fire perimeters from 3 APIs + ~15,000 USGS elevation calls + ~600 LANDFIRE EVT calls. All geological constants.

**New architecture:**
- `build_sites.py` — run manually to rebuild the catalog. Fetches fires, dedupes, enriches with elevation/slope/EVT, writes `data/sites.json` (388KB, 723 sites). Checked into git.
- `morel_finder.py` — reads `data/sites.json`, only fetches weather per-run. Much faster, no USGS hammering.
- `build_sites.py --update` — only enriches new sites not already in catalog.

### Wildfires included
Removed the `is_rx` filter. 95 wildfire sites now scored alongside 628 RX burns.

### Elevation caching
Changed from 24h TTL to permanent cache for successful lookups. Failed lookups (USGS 504s) retry after 1 hour.

*File: `utils/elevation.py`*

### Recency curve adjusted
Based on FR-002/003 (4-5lbs at 18mo burn), raised 15-20mo bracket from 0.20 → 0.40.

*File: `config.py:131-138`*

### GH Action simplified
Removed `actions/cache` step for API responses (no longer needed).

*File: `.github/workflows/daily-score.yml`*
